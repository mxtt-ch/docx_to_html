本项目是为了实现实现将docx文件转为html文件，要求html要实现原docx文件内容的显示、布局、样式、功能。

测试使用的docx文件是：ScholarHub数智服务平台使用指南（2025）.docx

脚本目录结构：
```
.
├── docx_to_html.py：提供项目入口main方法，和一些参数配置
├── convert_docx_to_html.py：提供将docx转为html文件的DocxToHTMLConverter核心类
├── uitl.py：提供一些工具函数
```

脚本生成结果：
```
output_时间戳 文件夹
├── images 文件夹 保存递归遍docx文件dom树时生成的所有图片
├── xml 文件夹 保存docx的所有xml文件
├── convert_log 文件 转换日志文件
├── guide.html 文件 转换后的html文件
```

需要实现功能：
1. 遍历docx的dom树，识别出所有的元素类型，比如段落、图片、表格等。
2. 针对不同的元素类型，实现对应的转换逻辑。
3. 根据不同的元素类型，生成对应的html标签。
4. 识别目录的相关内容(页码忽略、标题与页码的连接符忽略，仅保留序号和标题)，如果docx文件目录标题实现点击目录内的标题跳转到对应的内容标题，那么html文件目录标题跳转功能也需要实现。
5. 识别文本框与图片混排的效果，需要将文本框与图片混排的效果，且需要保持文本框与图片的位置关系。
6. 识别编号、项目符号的识别，需要编号、项目符号保持原有的格式，比如 序号的数据、字号、字体、颜色、与后面文字的间隔。
7. 识别内容超链接，需要支持点击跳转，且需要保持原本的样式关系。
8. 识别表格，需要将表格转换为html的table标签，且需要保持原本的样式关系。
9. 识别图片，需要将图片转换为html的img标签，且需要保持原本的样式关系，比如 图片的宽度、高度、与文字的间隔等，还需要出图片的特殊情况，当图片上面存在标注、文本框等其他内容，则需要将这些内容压到图片上在进行保存，且需要将图片保存到images文件夹内。
10. 识别文本框，需要将文本框转换为html的div标签，且需要保持原本的样式关系，比如 文本框的宽度、高度、与文字的间隔等。


css要求：
1. body 标签需要写为“width:1200px; margin: 0 auto;”
2. 其他所有css样式都必须基于docx文件的xml样式，这些css样式请写到标签的style属性内，且不要使用写死的静态样式，请从xml中读取转为css。
3. head 的 style 标签 不要写入任何css样式。
4. head 的 title 标签 优先使用传入的title参数，若未传入title参数，则写为“ScholarHub数智服务平台使用指南（2025）”

额外要求：
1. 项目在处理docx文件时 需要在cmd 中显示一个进度条，来展示转换进度，进度条需要实时更新，且该进度条需要独立不被其他代码所影响，比如：可能出现的 进度条多次渲染的问题。
2. 可以适当优化项目结构
3. 项目代码一定不能出错，请务必测试，不能出现任何异常情况。

解决bug时要求：
1. 必须结合ScholarHub数智服务平台使用指南（2025）.docx、convert_log、xml文件、html文件，进行深入的分析。
2. 我们要实现的功能是：将docx文件转为html文件，要求html要实现原docx文件内容的显示、布局、样式、功能，所以你需要对照docx文件、html文件阅读，分析出问题所在。
3. 日志文件 convert_log 中记录了转换过程中出现的日志，你需要根据日志文件分析出问题所在。
